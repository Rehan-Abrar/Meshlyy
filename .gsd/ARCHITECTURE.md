## Components

### Backend (Express)
- **Purpose:** Handles business logic, AI matching orchestration, and data persistence via Supabase.
- **Location:** `Backend/`
- **Key Modules:**
  - `routes/`: API endpoint definitions (onboarding, health, etc.).
  - `middleware/`: Auth verification, error handling, rate limiting.
  - `services/`: AI matching logic, integrations (Supabase, Cloudinary).
  - `lib/`: Shared utilities and custom error classes.

### Frontend (React)
- **Purpose:** User interface for brands and creators.
- **Location:** `Frontend/`
- **Key Modules:**
  - `features/`: Domain-specific components (admin, brand, influencer).
  - `components/`: Reusable UI elements.
  - `layouts/`: Page containers and navigation structures.

## Data Flow

1. **Authentication:** Users authenticate via Supabase Auth. The frontend receives a JWT.
2. **API Requests:** Frontend sends requests to the Backend with the JWT in the Authorization header.
3. **Processing:** Backend verifies the token (currently using a placeholder, JWKS integration TODO), processes logic, and interacts with third-party services (Gemini, Supabase, Cloudinary).
4. **Response:** Structured JSON responses are sent back to the frontend.
